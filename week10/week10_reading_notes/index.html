Fetch, form Validation

week10-notes and reading

Web API or Application Program Interface is a type of program that has data or information stored that
can be interacted with to be used for your sites on purposes. These interfaces are developed by a third
party and often involve large corporations who must use large amounts of information in a certain format,
with the format changing. Here are two ways we interact with api’s

  1. First, we have client-side form validation. In this instance we have an information field that we can
    fill out, if the information field doesn’t match up. Then we receive an error message. This is useful
    because it allows us to interact with the api and pull only information that is valid not information
    that is in valid.
  2. Next we also have fetch key word, this allows us to pull information from different Api’s sources then
    use them to fill out our page with data coming from them. these are very useful when we have a large
    database meant for storing information that we want to grab in a organized format.

All in all, the role of API’s are very important, they make it so we don’t have to constantly
update many different website we are using, we only need to make sure the api we are pulling from
is functioning correctly. And then only the api itself needs to be updated. Let’s say you have 20
different index.html pages. Instead of having to go through change every single on of those pages.
All you need to do is make sure the api is up to date and working.
