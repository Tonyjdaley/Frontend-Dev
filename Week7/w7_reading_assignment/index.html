This week we went into further understanding of functions, and we are learning
about what ajax files are. In truth I feel as though I wasn’t quite able to dig
much deeper with functions, they are just such a large topic. That it is still hard
for me to gather complete their depth and capabilities. Ajax however is very interesting
they are a form of file pathing it seems that allows background information to continue
running

  1. First off is Functions, as already known, function I see has the brain of our
    applications, they are what makes it think and process. That being said we continue
    to learn about the scope of what we can really accomplish with these functions. In
    truth its hard to name all the capabilities, but one that I liked and ill share is
    a promise it is a type of function that represents a future result, these are
    interesting to me because unlike most function they don’t create a immediate response,
    it is not necessarily a delayed response either. It is more something that will occur
    once the parameters are matched. This is cool because it allows you to attach multiple
    function onto a function and when parameters are met they begin to rise in scale.
  2. Ajax is very intriguing since it Is a way to make a web page not static, allowing for
    background information. This is revolutionary since being able to do it will allow
    updates on a site without actually having to reset it. An example of this might be a
    message board, lets say you message someone and then they message you back. If this page
    was static, it would then have to be refreshed in order for you to see the message. However,
    with the ability to update in the background we are essentially making it so it so it is always
    active and able to change as you make changes. This in itself is huge!
