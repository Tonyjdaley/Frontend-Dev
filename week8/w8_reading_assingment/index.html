CSS3 Tranisitions,Transforms, Animations. Canvas API, and SVG

week08-notes and reading

This week’s reading was spent learning how to alter elements in different ways using JS.
It also gave us understanding on how to pull arrays or information from other sources using
json files. Finally we learned a little bit about different types of API or formatting methods
that help increase quality of source images and such.

  1. Transforms, a are a more advance set of CSS rules (CSS3 that allow us greater control of
    the page elements.
  2. Transition is a type of CSS3 rule that allows us gradually to change from on state to
    another. For example, a hover effect instead of being immediate we can make the effect drawn
    out and slower in its change
  3. Animation is like Transition but use a frame state and moves between them to create a moving
    image or process.
  4. Json files are place holder files that can be easily translated or converted over the web
    and used in our applications.
  5. Canvas API is a flexible Application Programming interface that allows us to create any type
    of visual effect we can think of all through JS
  6. SVG or Scalable Vector Graphics, is a file format type that allows us to scale imagery and other
    type of visuals in our application with out losing Quality of the image itself. This is done
    using XML files
  7. XML files are a type of file format that allows easy transfer over the wire and become assets
    that can be used over the web.

All in all, this week had some really good insight, and we went pretty deep into how to use
resources, elements, assets, and such, that are not located in our index.html’s folders but
instead located across the wire in various location as needed. We then looked at different ways
to alter these data sets and how to obtain greater control over what these resources can do.
